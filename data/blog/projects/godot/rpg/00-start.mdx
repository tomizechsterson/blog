Let's see how far this Godot rabbit hole can take us. Now that Tomato Man is in a good place, it's time to switch gears and
try a different genre. I'm a big fan of RPG's, and I've even played some of the super old-school titles from back in the day.
I can only remember the early Ultima series at this point, but I'm sure I've played others...[^1] So how hard can it be to
make one of those? I mean, those a super primitive games according to our standards, so surly I can slap one together, right.
..? Right?? Who cares? Let's just try.

The idea will be to learn some more Godot things and see if we can push the scope a bit further than with Tomato Man. Now,
with an RPG, we'll need to focus on different features of the engine, such as UI bits, different tile-based features,
resources, and possibly even exploring BB Code and/or a dialogue plugin.[^2]

[^1]: It's been a very, very, _very_ long time, ok? Gimmie a break!

[^2]:
    A tutorial from a long time ago used a really neat plugin, so I'll have to go back and see if it's still around.
    Either way, I have no doubt there's a good plugin to try.
